# Archive

This folder contains **completed projects, inactive areas, and outdated resources** that you want to keep for reference but are no longer active.

## What Goes Here

- Completed projects (after closeout)
- Inactive areas no longer relevant
- Outdated resources and templates
- Historical records
- Lessons learned and retrospectives

## What Doesn't Go Here

- Active projects (keep in **Projects**)
- Current areas of responsibility (keep in **Areas**)
- Current resources (keep in **Resources**)

## Purpose

The Archive serves multiple purposes:

### Historical Reference
- Look back at past work
- Review decisions and outcomes
- Find examples for proposals
- Track business evolution

### Legal & Compliance
- Maintain records as required
- Support in case of disputes
- Tax documentation
- Contract history

### Learning & Growth
- Review what worked/didn't work
- Mine for insights
- Track personal growth
- Identify patterns

## Archiving Process

### For Projects

When a project is complete:

1. **Closeout Review**
   - Complete project-closeout.md template
   - Gather final deliverables
   - Collect client feedback
   - Document lessons learned

2. **Organization**
   - Move to Archive/projects/[Year]/[Quarter]/
   - Ensure all files are included
   - Update links and references
   - Add to archive index

3. **Cleanup**
   - Remove from active projects list
   - Update portfolio if showcase-worthy
   - Store deliverables securely
   - Update financial records

### For Areas

When an area becomes inactive:

1. **Review**
   - Why is it becoming inactive?
   - Any final actions needed?
   - Extract useful resources
   - Document outcomes

2. **Archive**
   - Move to Archive/areas/
   - Keep structure intact
   - Add archive date to README
   - Update area list

### For Resources

When resources become outdated:

1. **Evaluate**
   - Why is it outdated?
   - Any replacement available?
   - Historical value?

2. **Archive or Delete**
   - Move to Archive/resources/ if valuable
   - Delete if no future value
   - Update resource links

## Retention Policy

### Keep Forever
- Final project deliverables
- Signed contracts
- Tax documents
- Client testimonials
- Major milestones

### Keep 7 Years
- Financial records
- Invoices and payments
- Business expenses
- Tax-related documents

### Keep 3 Years
- Project files and code
- Communication records
- Meeting notes
- General business records

### Review Annually
- Old resources
- Inactive areas
- Outdated materials

## Archive Maintenance

### Annual Review
- Review archived items
- Delete per retention policy
- Update archive organization
- Extract any insights

### Quarterly Cleanup
- Archive completed projects
- Remove clutter from active areas
- Update archive index

## Access & Search

### Archive Index
Maintain an index of archived items:
- Project name
- Client name
- Date range
- Location
- Key tags

### Search Tips
- Use consistent naming
- Add metadata/tags
- Keep README files
- Maintain directory structure
