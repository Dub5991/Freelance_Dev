## Purpose
Weekly planning and prioritization based on quarter goals.

## Structure
Create one file per week: `Week-2026-01.md`, `Week-2026-02.md`, etc.

## What Goes Here
- **Top 3 Priorities** - Most important outcomes for the week
- **Client Deliverables** - What needs to ship this week
- **Internal Work** - Admin, learning, business dev
- **Time Blocks** - Rough schedule of focus time
- **Wins** - End-of-week celebration section

## Template Structure
```markdown
# Week 2026-01 (Jan 1-7)

## Top 3 Priorities
1. Complete [BILLABLE] task-20260102-001 for Acme Corp
2. Send Q4 2025 invoices and follow up on outstanding
3. Update strategic pillars and Q1 goals

## Client Work (40 hours)
- Acme Corp: 20 hours - Feature deployment
- Tech Startup: 10 hours - Architecture review
- Legacy Inc: 10 hours - Bug fixes

## Internal Work (10 hours)
- [ ] Weekly review and planning (2h)
- [ ] Update client health scores (1h)
- [ ] Record expenses for December (1h)
- [ ] Portfolio update with recent projects (2h)
- [ ] Learn new AWS service (4h)

## Time Blocks
- **Mon-Wed**: Deep work on Acme deliverable
- **Thu**: Client meetings and check-ins
- **Fri**: Internal work and learning

## Wins ðŸŽ‰
_Fill this out Friday afternoon_
```

## Best Practices
- Plan on **Friday afternoon** or **Monday morning**
- Keep priorities to **3 max** for focus
- Block time for deep work on priorities
- Review and celebrate wins on Friday

## MCP Integration
- **work_server** - Pull P0/P1 tasks for the week
- **client_server** - Check for upcoming meetings
- **billing_server** - See outstanding invoices
