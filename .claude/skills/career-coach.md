# Career Coach Skill

## Description
Plan career development, track skills, set learning goals, and manage professional growth using the Career Server.

## When to Use This Skill
- Assessing current skills
- Identifying skill gaps
- Setting learning goals
- Building portfolio
- Planning rate increases
- Growing professional network
- Tracking career milestones

## MCP Servers Used
- **career_server**: Primary server for professional development
- **work_server**: For tracking project experience
- **billing_server**: For rate history

## Step-by-Step Instructions

### 1. Skills Inventory
```
1. Add skills using add_skill()
2. Set proficiency level (1-5)
3. Track years of experience
4. Update last used date
5. Review inventory regularly
```

**Example Prompts:**
- "Add Python skill at expert level with 5 years experience"
- "Update my React proficiency to level 4"
- "Show my complete skills inventory"

**Proficiency Levels:**
- 1: Beginner - Learning basics
- 2: Intermediate - Can work with guidance
- 3: Advanced - Independent work
- 4: Expert - Can teach others
- 5: Master - Industry recognized

### 2. Skill Gap Analysis
```
1. Define target role or capabilities
2. List required skills
3. Use identify_skill_gaps()
4. Review missing skills
5. Create learning plan
```

**Example Prompts:**
- "Analyze skill gaps for senior full-stack developer role"
- "What skills do I need for DevOps engineering?"

### 3. Learning Goals
```
1. Create goals using create_learning_goal()
2. Set target proficiency
3. Define timeline
4. List resources
5. Track progress with update_learning_progress()
```

**Example Prompts:**
- "Create learning goal to master TypeScript in 90 days"
- "Update progress on AWS certification to 75%"
- "Show all active learning goals"

### 4. Portfolio Management
```
1. Add projects using add_portfolio_project()
2. Include technologies used
3. Add live URL and GitHub link
4. Highlight key achievements
5. Feature best projects
```

**Example Prompts:**
- "Add e-commerce platform project to portfolio"
- "List all portfolio projects using React"

### 5. Rate Planning
```
1. Record rate changes using record_rate_change()
2. Track growth over time
3. Research market rates
4. Plan annual increases
5. Justify rate with value
```

**Example Prompts:**
- "Record rate increase to $175/hour effective next month"
- "Show my rate history"
- "What's been my rate growth percentage?"

### 6. Network Building
```
1. Add contacts using add_network_contact()
2. Categorize relationships
3. Track last contact
4. Plan regular touchpoints
5. Nurture referral sources
```

**Example Prompts:**
- "Add John Smith as a mentor contact"
- "List all referral source contacts"
- "Show network contacts I haven't talked to in 90 days"

### 7. Milestone Tracking
```
1. Record achievements using add_milestone()
2. Celebrate wins
3. Document impact
4. Use in portfolio/resume
5. Share on LinkedIn
```

**Example Prompts:**
- "Add milestone for reaching $100K revenue"
- "Record AWS certification completion"

## Skill Development Strategy

### T-Shaped Skills
- **Depth**: Deep expertise in 1-2 areas
- **Breadth**: Working knowledge across many areas

### Learning Priorities:
1. **Core Competency** - Your main expertise (80%)
2. **Adjacent Skills** - Complementary capabilities (15%)
3. **Emerging Tech** - Future opportunities (5%)

### Learning Time Budget:
- **Daily**: 30-60 minutes
- **Weekly**: 5-10 hours
- **Monthly**: 20-40 hours
- **Annual**: 240-480 hours (10-20% of time)

## Rate Increase Strategy

### When to Raise Rates:
- After skill improvements
- Strong demand for services
- Positive client feedback
- Market research supports it
- Annually at minimum

### How Much to Increase:
- **Annual**: 5-10% cost of living
- **Skill-based**: 10-20% for new capabilities
- **Market-based**: Match market rates
- **Value-based**: Based on client ROI

### How to Communicate:
```
"Effective [Date], my rates will be $[New Rate]/hour. 
This reflects my continued investment in [skills/tools]
and the increased value I provide. Current projects 
will maintain existing rates through completion."
```

### Grandfather Existing Clients:
- Option 1: Immediate for new projects only
- Option 2: Phase in over 3-6 months
- Option 3: Apply to all work after notice period

## Portfolio Best Practices

### Project Selection:
- ✅ Show diverse capabilities
- ✅ Highlight impact and results
- ✅ Include recent work (last 2 years)
- ✅ Demonstrate problem-solving
- ✅ Show technical depth

### What to Include:
- Project name and description
- Your role and responsibilities
- Technologies used
- Key challenges solved
- Results achieved
- Client testimonial (if available)
- Live link or demo
- Code samples (if allowed)

### Case Study Structure:
1. **Challenge** - What was the problem?
2. **Approach** - How did you solve it?
3. **Solution** - What did you build?
4. **Impact** - What were the results?
5. **Technologies** - What did you use?

## Error Handling

**"Skill already exists"**
- Update existing skill instead
- Check spelling and case

**"Invalid proficiency level"**
- Use 1-5 scale only
- Be honest in self-assessment
